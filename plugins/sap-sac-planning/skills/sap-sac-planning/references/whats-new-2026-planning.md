# What's New in SAP Analytics Cloud Planning — 2026

## Overview

This reference covers planning-relevant features introduced in SAC QRC1 2026 (version 2026.3) and QRC2 2026 (version 2026.8). Each entry describes the feature, its planning use case, and how to use it. All features are generally available unless noted.

---

## QRC1 2026 (2026.3)

### AI-Assisted Data Actions

In the Script tab of advanced formula steps in data actions, you can use AI-assisted data actions to generate scripts from natural-language comments, or generate explanatory comments from existing scripts.

**How to use it**:
1. Open a data action with an advanced formula step.
2. In the Script tab, write a comment describing what the script should do (e.g., `// Copy actuals to budget and apply 10% growth`).
3. Select the comment and click **Generate Script**. The AI generates a script below the comment with an "Accept/Reject" prompt.
4. Verify the generated script before accepting. Lines generated by AI are highlighted with vertical markers.

**Generating comments**: Select an existing script block and click **Generate Comment** to produce an AI-generated explanation. Useful for documenting inherited or complex formulas.

**Prerequisites**: The Execute permission for the Generative AI privilege must be assigned. AI-assisted features must be enabled by an administrator.

**Constraints**:
- When referring to members in comments, use double quotes around IDs (e.g., `"BUDGET_2026"`) for best accuracy.
- If members with the same ID exist across multiple dimensions, include the dimension name in the comment.
- Do not use a previously generated comment as the basis for script generation — it may omit technical details like filters and calculations.

Source: https://help.sap.com/docs/SAP_ANALYTICS_CLOUD/00f68c2e08b941f081002fd3691d86a7/3f95d6d495e140bfaf357400854ab678.html

### Live Data Connectivity to Snowflake for Planning Models

SAC now supports live data connections to Snowflake for planning-capable models. This allows planners to work directly with Snowflake data as a live external version, reducing data loading overhead and simplifying model configuration.

**How to use it**:
1. Enable the Data Access Agent for Snowflake connectivity.
2. Create a new planning model and select the Snowflake source as a live connection.
3. Configure the model using the same modeling flexibility available for existing live connections (e.g., Google BigQuery).
4. Plan normally — data entry and version management work against the live Snowflake source.

**Planning relevance**: Reduces total cost of ownership for organizations with Snowflake data warehouses by eliminating the need to replicate data into SAC storage before planning.

Source: https://community.sap.com/t5/technology-blog-posts-by-sap/sneak-peek-in-to-sap-analytics-cloud-release-for-q1-2026/ba-p/14317657

### Story Versioning

Story designers can now create, manage, and restore up to ten major versions of a story. This provides greater control over iterative development of planning dashboards and analytical applications.

**How to use it**:
1. Open a story in designer mode.
2. Access version history from the story menu.
3. Open previous versions in read-only format, restore them, or save them as new artifacts.

**Planning relevance**: Enables safe experimentation with planning story layouts, table configurations, and dashboard designs without risking published content. This is the foundational phase of a larger multi-artifact versioning roadmap.

**Limitation**: Version history is lost when a story is copied or saved as a new file.

Source: https://community.sap.com/t5/technology-blog-posts-by-sap/sneak-peek-in-to-sap-analytics-cloud-release-for-q1-2026/ba-p/14317657

## QRC2 2026 (2026.8)

### Asymmetric Reporting

Asymmetric reporting enables the design of SAC tables with differing time ranges, hierarchies, and measures per row or column. This is particularly important for finance-style profit and loss views, combined actuals and forecasts, and highly flexible business reporting.

**How to use it**:
1. In a story table based on a planning model, use visibility filters for table widgets.
2. Apply distinct visibility filters per structure member.
3. Hide unwanted parent levels across an entire hierarchical dimension.
4. Define exactly which hierarchy levels are visible — the system automatically expands to the highest selected level.
5. Show or hide total rows and columns per structure member.
6. Choose specific members to display for flat dimensions and navigational attributes.

**Planning relevance**: Asymmetric tables work in planning models and support data entry. Planners can create finance-style P&L layouts with mixed actuals and forecast columns, each with different time granularity.

**Key capabilities**:
- Measure Input Control in Restricted Measures and Cross Calculations
- Dynamic Time Filters for "Rest of Period"
- Visibility Filters for Table Widgets

Source: https://community.sap.com/t5/technology-blog-posts-by-sap/sneak-peek-in-to-sap-analytics-cloud-release-for-q2-2026/ba-p/14384997

### Composite Versioning

Composite designers can now manage up to 10 versions of a composite during story design. This enables safe iteration, easy rollback, and consistent reuse across stories.

**How to use it**:
1. Open a composite in the designer.
2. Access version history to view all saved versions.
3. Restore or revert to a previous version when needed.
4. Only one active version can be used in a story at a time.

**Planning relevance**: Reduces disruption for story viewers by allowing quick recovery to a stable composite version. Improves reliability and safety when updating composite artifacts used in planning dashboards.

Source: https://community.sap.com/t5/technology-blog-posts-by-sap/sneak-peek-in-to-sap-analytics-cloud-release-for-q2-2026/ba-p/14384997

### Data Export API in Job Monitor

A new **Data Export API** tab has been introduced in the job monitor, allowing you to check the status of data extraction and delta calculation jobs that use the Data Export Service.

**How to use it**:
1. Open **Job Monitor** from the SAC toolbar.
2. Switch to the **Data Export API** tab.
3. View status and progress of full and delta export jobs.
4. Troubleshoot faster with visibility into job states.

**Planning relevance**: Provides enhanced visibility for planning data orchestration scenarios where SAC exports plan data to external systems (e.g., S/4HANA write-back integrations).

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US

### Data Import Service API: Master Data into Datasphere Public Dimensions

The Data Import Service API now supports importing master data into public dimensions stored in SAP Datasphere. This extends the API beyond SAC-stored dimensions to include Datasphere-managed dimensions used in seamless planning models.

**How to use it**:
1. Use the Data Import Service API endpoint with the target dimension set to a Datasphere-stored public dimension.
2. Structure the import payload with the dimension members and properties.
3. Execute the import — master data is written directly to Datasphere.

**Planning relevance**: Critical for seamless planning workflows where master data governance resides in Datasphere. Enables programmatic master data maintenance without switching tools.

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US

### Data Import Service API: External Fact Data to Seamless Planning Private Versions

The Data Import Service API now lets you import external fact data — from sources other than SAC and Datasphere — into an existing private version of a seamless planning model.

**How to use it**:
1. Create or identify a private version in a seamless planning model.
2. Use the Data Import Service API to import fact data from an external source.
3. The data is written to the specified private version for validation before publishing.

**Planning relevance**: Enables loading external plan data (e.g., from third-party planning tools, market data providers) directly into SAC private versions for comparison and what-if analysis before publishing to public versions.

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US

### onAfterExecute Event Enhancement

The `onAfterExecute` event for data uploads now provides additional information beyond the status. This event is commonly used in Analytics Designer planning applications.

**New fields available**:
- **Message**: Description of the upload result.
- **Statistics**: Rows processed and other performance metrics.
- **Rejected records**: List of records that failed validation.
- **Target version**: Name of the version data was uploaded to.
- **File name**: The file that was uploaded.

**Planning relevance**: Improves error handling in custom planning applications by providing granular feedback on data upload operations.

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US

### Multi-Action API Step: HTTP 204 Response

HTTP response 204 (No Content) has been added to the list of expected results for multi-action API steps. This response indicates the server successfully processed the request but has no content to return.

**How to use it**: When configuring a multi-action API step, HTTP 204 is now accepted as a success response alongside existing status codes.

**Planning relevance**: Fixes integration scenarios where REST APIs return 204 on successful execution (common for delete, update, or trigger operations in planning workflows).

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US

### Enhanced HTTP Header Restrictions for Multi-Action API Steps

The restrictions on HTTP header field names for multi-action API steps have been enhanced. This improves security posture when calling external APIs from planning workflows.

**How to use it**: When configuring headers in multi-action API steps, follow the updated character restrictions documented in the SAP Help Portal.

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US

### Calendar: Team References Retained Upon Event Activation

When activating a calendar event, the system now keeps team references by default instead of automatically resolving them to individual users. If team membership changes, the event owner receives a warning and can synchronize teams.

**How to use it**:
1. Create or activate a calendar event (general task, review task, composite task) involving teams.
2. If team membership has changed, a warning appears in the Details panel.
3. Synchronize individual teams or all teams involved in the event.
4. Use the Edit Events wizard to synchronize teams across multiple events.

**Planning relevance**: Reduces maintenance overhead for recurring planning cycles where team membership changes between cycles. Automatic synchronization also occurs when reopening events, updating events, or when a review task assignee rejects work.

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US

### Automatic Loading Smart Insights Top Contributors

For SAP Datasphere and Seamless Planning models, progressive loading of Top Contributors is now enabled automatically when the number of dimensions or hierarchy levels exceeds 20. No configuration is required.

**Constraints**: Top Contributors are not supported for non-additive measures (e.g., ratios or exception aggregations).

**Planning relevance**: Improves the Smart Insights experience for large seamless planning models without manual tuning.

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US

### Mixing Advanced Filters in Just Ask

Just Ask now supports mixing excluded date members with date range filters. For example: `Show me taxes by product for 2013–2017 excluding Q3 2015`. The exclusion is preserved by applying an Advanced Filter.

**Planning relevance**: Enhances ad-hoc analysis during planning meetings when users need to exclude specific periods from their queries.

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US

### Deprecation: Export Model Data to SAP S/4HANA

Exporting model data to SAP S/4HANA is now deprecated as of Q2 2026. SAP recommends using the write-back integration scenario instead. Refer to SAP Note 3707288 for migration guidance.

**Planning relevance**: Plan data export workflows should migrate to the write-back integration. Existing export jobs continue to function but will not receive future enhancements.

Source: https://help.sap.com/whats-new/42e4f84a0e5e458792b1047eaf81c31a?locale=en-US
