## Why this matters

The repo's AI coverage has a language gap. `sap-ai-core` covers the platform
(REST, orchestration concepts, GenAI Hub) and `sap-cloud-sdk-ai` covers the
JavaScript/TypeScript and Java SDKs — but nothing covers **Python**. SAP's
official Python path is the **`sap-ai-sdk-gen`** PyPI package ("SAP Cloud SDK
for AI (Python): generative AI SDK", v6.10.0 at planning time). Note the
history: the previous package `generative-ai-hub-sdk` is **deprecated** (its
PyPI page says "no longer maintained, use sap-ai-sdk-gen instead" — verified
2026-06-11); most existing tutorials and user vocabulary still use the old
name, which makes skill guidance on the rename extra valuable. Python is the
dominant language of the data-science audience this marketplace already
serves (`sap-hana-ml`, SAC, Datasphere users). Today a user asking the AI
assistant for Python GenAI-Hub code gets no skill support at all.

access (beyond pip) can use the skill successfully.

## Steps

### Step 1: Research and verify the SDK surface

Do not rely on memory. Primary method is **local introspection** (no JS
needed): create a temp venv OUTSIDE the repo and install the SDK
(introspection command in the table above), then enumerate the real module
tree and signatures, e.g.:

```bash
/tmp/sdkcheck/bin/python - <<'EOF'
import pkgutil, gen_ai_hub
for m in pkgutil.iter_modules(gen_ai_hub.__path__):
    print(m.name)
EOF
```

and `inspect.signature(...)` / `help(...)` on the entry points you will
document. Cross-check usage patterns against the SAP-samples notebooks (plain
fetch) and, where available, the help.sap.com SDK reference (firecrawl).

Confirm:

1. Current `sap-ai-sdk-gen` version on PyPI (command in table) and the
   install extras (`[all]`, `[google, amazon]`, … — listed in the PyPI README).
2. The package's main modules and entry points. Expected based on the
   predecessor (verify each via introspection — the import name is likely
   still `gen_ai_hub`): proxy/native client integrations (OpenAI-compatible,
   Amazon, Google), LangChain integration (`init_llm`,
   `init_embedding_model` or current equivalents), and the orchestration
   client (templating, LLM config, content filtering, data masking,
   grounding).
3. Authentication: AI Core service key via `AICORE_*` environment variables
   and/or `~/.aicore/config.json` profiles — confirm exact variable names
   from the SDK source/docstrings or docs.
4. The supported-models statement (point to SAP's model catalog rather than
   hardcoding an exhaustive list).

**Verify**: for each of the four items you have the verified names (from
introspection) plus at least one source (PyPI README, sample notebook, or
scraped doc page). Clean up the venv afterwards (`rm -rf /tmp/sdkcheck`). If
`sap-ai-sdk-gen` is itself deprecated or superseded on PyPI, STOP.
