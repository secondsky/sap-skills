## Research tooling note (read before executing any plan)

**SAP web properties are not plain-fetchable.** Verified by probe 2026-06-11:
help.sap.com `/docs` and `/whats-new` pages and sap.com pages are JS-rendered
SPAs (plain HTTP fetch returns an empty shell — and HTTP 200 status proves
nothing); community.sap.com returns 403 to plain HTTP clients; SAP KBA pages
are JS-rendered and may be login-gated. JS-rendered sources must be read with
a JS-rendering scraper — in this environment the Firecrawl MCP tools
(`mcp__mcp-server-firecrawl__firecrawl_scrape` with `formats: ["markdown"]`,
`mcp__mcp-server-firecrawl__firecrawl_search`). Each plan carries a
per-source access-method table and a STOP condition for the no-scraper case.

JS-free sources that DO work with plain fetch (verified 2026-06-11):

- `github.com/SAP-docs/sap-datasphere` — markdown mirror of Datasphere help (plan 001 follow-ups)
- `github.com/SAP-docs/btp-cloud-identity-services` and
  `github.com/SAP-docs/btp-cloud-identity-services-system-integration-guide` — IAS docs mirrors (plan 006)
- `github.com/SAP-samples/btp-gen-ai-hub-sdk-samples` and related sample repos (plans 005, 007)
- `https://pypi.org/pypi/<package>/json` — PyPI metadata + README (plan 005)
- The SAC Custom Widget Developer Guide PDF (direct `application/pdf` download; plan 003)
- pip install + Python introspection in a temp venv outside the repo (plan 005)

**No GitHub markdown mirror exists for SAC or AI Core documentation** — plans
002/003/004/007 require a Firecrawl-class tool.

**Package rename caught during revision**: `generative-ai-hub-sdk` on PyPI is
deprecated; the successor is `sap-ai-sdk-gen` ("SAP Cloud SDK for AI
(Python)", v6.10.0 at revision time). Plan 005 targets the successor. Future
plans touching the Python AI SDK must check the PyPI deprecation banner first.

**Skill content rule**: the finished skills are consumed by AI harnesses with
the same fetch constraints — every reference `.md` must inline the actual
knowledge (a "Self-contained knowledge rule" section in each plan); URLs are
citations only, never the substance.

## Findings considered and rejected

So nobody re-audits these:

- **`__pycache__/*.pyc` files under four plugins' `hooks/` dirs**: untracked
  and ignored (`.gitignore:1` has `__pycache__/`); local build artifacts, not
  a repo hygiene issue.
- **plugin.json `version: 2.2.2` differing from SKILL.md `metadata.version`**:
  by design — plugin.json carries the repo-global marketplace version managed
  by `sync-plugins.sh`; `metadata.version` is the per-skill content version.
- **Standalone `sap-business-data-cloud` plugin**: deferred, not rejected.
  BDC's developer-facing surface beyond Datasphere/SAC is still thin; plan 001
  folds BDC context into sap-datasphere instead. Revisit when BDC exposes a
  substantial developer API surface.
- **Standalone Generative AI Hub plugin**: rejected by maintainer decision
  2026-06-11 — it would duplicate sap-ai-core's ~2,000 lines of GenAI Hub
  content (`generative-ai-hub.md`, `orchestration-modules.md`,
  `model-providers.md`, `grounding-rag.md`). Plan 007 enriches sap-ai-core
  instead.
- **New BTP skill for Event Mesh / Advanced Event Mesh or Kyma**: deferred.
  Event Mesh is partially covered inside `sap-btp-integration-suite` (it is an
  Integration Suite capability); Kyma appears across 17 plugins as a
  deployment target. Cloud Identity Services (plan 006) was judged the higher-
  leverage gap (identity mentioned in 15+ plugins, owned by none).
- **sap-sac-scripting `template_patterns: 56` metadata**: unverified count but
  no evidence it's wrong; plan 004 has a STOP condition if it's off by a wide
  margin during execution.
