## Why this matters

Identity is the largest cross-cutting gap in the marketplace's 14-plugin BTP
category: `grep -ril "cloud identity\|identity authentication\|xsuaa" plugins/`
matches files in 15+ plugins (connectivity, CAP, work zone, integration suite,
cloud logging, …), yet no plugin owns the topic. Every productive BTP
application needs authentication and authorization, and SAP is actively
migrating the platform from XSUAA toward SAP Cloud Identity Services
(Identity Authentication — IAS, Identity Provisioning — IPS, Authorization
Management — AMS). Users currently get fragmented, secondhand identity
guidance scattered across skills whose actual subject is something else.

## Self-contained knowledge rule

The finished skill is consumed by AI harnesses that cannot fetch SAP URLs at
usage time. Every reference file must inline the actual guidance —
configuration walkthroughs, policy syntax, integration steps, error causes —
never defer substance to a URL ("see <URL>" is not a section body; URLs are
end-of-section citations). Acceptance test: a reader with no internet access
gets complete guidance from the .md alone.
- Sibling skills to read for boundaries (do not duplicate their content):
  - `plugins/sap-btp-connectivity/skills/.../SKILL.md` — destinations/connectivity (it touches auth flows; this new skill owns identity *services*, connectivity keeps destination auth mechanics)
  - `plugins/sap-cap-capire/` — CAP app-level auth usage stays there; this skill owns tenant/service configuration and migration
  - `plugins/sap-btp-best-practices/` — keeps high-level security governance
